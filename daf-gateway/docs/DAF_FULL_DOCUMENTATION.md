## 1. Концепция Dynamic Activation Fabric (DAF)

DAF — это **живая платформа знаний**, где любая нейросеть (Grok, Claude, GPT, Llama и др.) может:

- Мгновенно искать по 1719+ MD-документам (1.68 млн слов)
- Скачивать нужные файлы в свою песочницу для сверхбыстрой работы
- Записывать изменения обратно в GitHub через токен
- Работать с голосом (автоматическая очистка ошибок диктовки)
- Активировать динамические скрипты и все 13 MCP-серверов по запросу

**Ключевая идея:** Репозиторий — это не просто хранилище файлов, а **набор живых инструментов**, которые нейросеть может активировать.

## 4. Ключевые технические решения

### 4.1. Очистка голосового ввода
```python
def clean_voice_query(text: str) -> str:
    replacements = {
        "лоренцо": "Lorenzo", "наутилус": "Nautilus",
        "связь": "Svyazi", "протокол": "protocol"
    }
    for k, v in replacements.items():
        text = text.replace(k, v)
    return text.strip().lower()
```

### 4.2. Запись в GitHub
Используется `GITHUB_TOKEN` + `git commit + push` через subprocess.

### 4.3. Скачивание в песочницу
Нейросеть может запросить любой файл и получить его содержимое для локальной работы.

### 4.4. Активация MCP
Через `/v1/chat/completions` нейросеть автоматически вызывает нужные MCP-сервера (`mcp_search_server`, `mcp_runner_server` и др.).

## 5. Как использовать (для любой нейросети)

### Системный промпт (рекомендуется):
```
Ты — агент Dynamic Activation Fabric (DAF) репозитория info10.

У тебя есть доступ к:
- /v1/chat/completions — умный поиск
- /api/download — скачай файл в мою песочницу
- /api/git/write — запиши изменения в GitHub
- clean_voice_query — всегда очищай мой голосовой ввод

Если я говорю голосом — всегда применяй clean_voice_query.
Если нужно быстро работать с файлом — скачай его.
Если нужно изменить информацию — используй git_write.
```
